## Design Context

### Users

**Who**: Founders and operators managing 2-10 B2B SaaS startups simultaneously.

**Context**: Checking daily, usually in under 3 minutes. Already overwhelmed by Slack, email, investor updates, hiring. This is where they come to cut through the noise and answer one question: "Which startup needs attention today, and what should I do next?"

**Job to be done**: Rapid triage across a portfolio. Spot the broken funnel, the MRR drop, the stale data. Create a task. Move on. Not analysis -- decisions.

**Mental state when using**: Tired, scanning quickly, context-switching between startups. The UI must reduce cognitive load, not add to it.

### Brand Personality

**Three words**: Data-forward, Surgical, Calm

- **Data-forward**: Numbers over decoration. Signal over noise. Metrics are the primary visual element, not chrome around them.
- **Surgical**: Precise, intentional, zero fluff. This is not a playful tool. Every pixel earns its place.
- **Calm**: Founders are already overwhelmed. The UI should reduce stress, not add energy. Quiet confidence, not excitement.

**Balance point**: Drift too cold and you get enterprise BI. Drift too playful and you get a Notion/Linear clone. This product lives in the narrow band between -- warm enough to feel trustworthy, sharp enough to feel professional.

**Emotional goal**: The founder opens this and exhales. Clarity. Control. "I know where I stand."

### Aesthetic Direction

**Primary reference**: Luma -- but 20% more technical and 30% less soft.

**Keep from Luma**:
- Soft elevation (cards feel like surfaces, not boxes)
- Warm neutral palette (not blue-heavy SaaS default)
- Generous spacing and breathing room
- Subtle gradients for depth (not decoration)
- Low visual noise (few borders, more layering)

**Do NOT copy from Luma**:
- Consumer softness -- this needs more precision
- Rounded-everything UI -- keep some sharpness
- Event/social vibe -- this is analytical and operational

**Anti-references** (explicit traps to avoid):
1. **Generic admin dashboards** -- Bootstrap tables, hard borders everywhere, blue + gray monotony, "SaaS template #482"
2. **Notion** -- Too playful, too loose, not enough hierarchy for metrics
3. **Stripe dashboards** -- Too dense, too data-heavy without guidance, feels like tools not decisions
4. **Hacker dashboards (Grafana-style)** -- Raw, ugly, engineer-first, zero emotional design

**Theme**: Dark-first, light mode second. Both supported.

### Color System

**Base palette** (dark mode primary):
- Warm dark background (not pure black)
- Neutral surfaces (slightly lighter than bg)
- Soft contrast, not harsh

**Semantic colors** (these matter more than brand color):
- Success (green) -- growth, healthy
- Warning (amber) -- needs attention
- Danger (red) -- broken, urgent
- Info (blue/purple) -- neutral signals

**Accent** (VERY restrained):
- One subtle accent (muted blue or violet)
- Used sparingly: links, focus states, key actions only
- This product is not about brand expression -- it's about state visibility

### Typography

- Replace Inter with something distinctive but still highly legible for data
- Tabular numerals are critical (metrics must align)
- Type hierarchy must clearly separate: page title > section header > metric value > label > supporting text
- Body text must be comfortable at scanning speed

### Accessibility

- WCAG AA compliance
- Strong contrast enforced for: text, charts, alerts
- Color is never the only signal (always paired with icons + labels)
- Designed for: tired founders scanning quickly making decisions
- Readability > aesthetic purity

### Design Principles

1. **State over style**: The visual system exists to communicate startup health states. Every color, weight, and size choice should make state more legible, not more decorative.

2. **Calm clarity**: Reduce visual noise at every opportunity. Fewer borders, more whitespace, softer elevation. The interface should feel like a quiet room, not a control panel.

3. **Decision-ready**: Every screen answers a question. The hierarchy must make the answer visible in 2 seconds, with supporting detail available on demand (progressive disclosure).

4. **Earned complexity**: Show the simple answer first. Let the founder dig deeper only when they choose to. Health badge > north-star metric > supporting metrics > funnel > insight. Each layer is opt-in.

5. **Surgical precision**: No generic patterns. Every component is designed for its specific job. A health hero is not "a card with a big number" -- it's a purpose-built status surface.

### Implementation

- **Component library**: shadcn/ui
- **Styling approach**: Moving from inline CSS to shadcn's CSS-based approach
- **Dark mode**: Primary theme, using CSS variables for theme switching
- **Spacing**: Generous, Luma-inspired breathing room
- **Elevation**: Soft shadows and subtle background layering instead of borders
- **Border radius**: Moderate -- not fully rounded (too soft), not sharp (too cold). ~0.5rem-0.75rem range.
- **Motion**: Minimal, purposeful. No bounce, no elastic. Smooth ease-out for state transitions only.
